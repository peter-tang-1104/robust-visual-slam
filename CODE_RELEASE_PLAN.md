# 代码发布计划

## 当前状态

- ✅ **演示视频**：已公开，可通过GitHub Releases访问
- 🔒 **源代码**：暂不公开，等待论文接收

## 发布流程

### 阶段1：论文提交阶段（当前）

- [x] 创建GitHub仓库（私有）
- [x] 上传演示视频到GitHub Releases
- [x] 创建项目README和文档
- [ ] 论文提交

### 阶段2：论文审稿阶段

- [ ] 保持代码仓库为私有状态
- [ ] 演示视频保持公开，供审稿人查看
- [ ] 根据审稿意见完善代码和文档

### 阶段3：论文接收后

- [ ] 将GitHub仓库从Private改为Public
- [ ] 上传完整的源代码
- [ ] 更新README中的状态信息
- [ ] 添加详细的代码文档和使用说明
- [ ] 创建正式版本Release（包含代码和视频）

## 注意事项

1. **视频公开性**：即使仓库是私有的，GitHub Releases中的文件也可以通过直接链接访问，这样可以在代码未公开时先展示视频效果。

2. **代码保护**：在论文接收前，确保：
   - 仓库设置为Private
   - 不要将源代码推送到GitHub
   - 只在本地维护代码

3. **论文接收后**：
   - 将仓库改为Public
   - 推送完整源代码
   - 更新所有文档中的状态信息
   - 创建正式Release版本
